Document Ingestion Pipeline

Multi-source, multi-format ingestion with intelligent extraction and normalization.

Connectors

Supported Sources

  • SharePoint Online (MS Graph delta sync)
  • Azure Blob / ADLS Gen2 (event-driven)
  • OneDrive for Business
  • Azure SQL / Cosmos DB (CDC)
  • M365 Exchange / Teams (Compliance API)
  • REST API / Webhook push
  • SFTP / FTP (via ADF)
  • SAP / Salesforce (via Logic Apps)
Formats

Supported File Types

  • PDF (native text + OCR scanned)
  • DOCX / DOC / RTF
  • XLSX / CSV / TSV
  • PPTX / PPT
  • HTML / Markdown / Plain text
  • Images: PNG, JPEG, TIFF (OCR)
  • Email: MSG / EML
  • JSON / XML (structured)
Processing

Chunking Strategy

  • Semantic chunking (sentence boundaries)
  • Max chunk: 512 tokens, overlap: 51 tokens
  • Parent-child hierarchy preserved
  • Table/list structure-aware splitting
  • Section/heading metadata retained
  • Multi-modal: separate text + image chunks
  • Small-to-big retrieval supported
Enrichment

Metadata Enrichment

  • NER: People, orgs, locations, dates
  • PII detection + selective redaction
  • Sentiment + toxicity scoring
  • Document classification (custom model)
  • Language detection + translation flag
  • Keyword extraction (TF-IDF + KeyBERT)
  • Summarization (per chunk + per doc)
Quality

Data Quality Gates

  • Min content length: 50 characters
  • OCR confidence threshold: ≥ 0.7
  • Duplicate detection (MinHash LSH)
  • Near-duplicate suppression (Jaccard ≥ 0.9)
  • Virus scan: Defender for Storage
  • Schema validation before indexing
  • Dead-letter queue for failed docs
SLA

Performance Targets

  • Ingestion latency: < 30s per document
  • Batch throughput: 50K docs/hour
  • Embedding throughput: 5M tokens/min
  • Index update: near-real-time (< 60s)
  • Re-indexing: full rebuild < 4 hours
  • Availability: 99.9% ingestion pipeline

Vector Database Strategy

Hybrid search combining dense vectors, BM25 keyword, and semantic re-ranking on Azure AI Search.

Index Design

Azure AI Search Schema

  • Field: id (Edm.String, key)
  • Field: content (Edm.String, searchable)
  • Field: contentVector (Collection Float32, 3072d)
  • Field: title, source, tenantId, docType
  • Field: createdAt, updatedAt, page, section
  • Field: piiRedacted (Edm.Boolean)
  • Field: classificationLabel, confidenceScore
  • HNSW: efConstruction=400, m=4
Retrieval

Hybrid Search Pipeline

  • Dense vector: cosine similarity (HNSW)
  • Keyword: BM25 full-text search
  • Fusion: Reciprocal Rank Fusion (k=60)
  • Semantic re-rank: cross-encoder (top 50)
  • Query expansion: HyDE hypothetical doc
  • Multi-query: 3 LLM-generated variants
  • Filters: tenantId, docType, date range
  • Top-K: default 10, max 50
Embedding Model

Embedding Configuration

  • Primary: text-embedding-3-large (3072d)
  • Compressed: text-embedding-3-small (1536d)
  • Batch size: 2048 tokens per request
  • TPM quota: 1M tokens/min (PTU)
  • Caching: Redis TTL 1h for identical inputs
  • Multi-lingual: native 100+ language support
  • Matryoshka: dimension reduction supported
Multi-tenancy

Tenant Isolation Strategy

  • Index-per-tenant (high isolation, <100 tenants)
  • Filter-per-tenant (scale, 100+ tenants)
  • Hybrid: tiered by data sensitivity
  • RBAC: Row-level security via filter
  • Encryption: Per-tenant CMK in Key Vault
  • Quota: Per-tenant RU/search unit limits
Performance

Search Performance Targets

  • P50 query latency: < 50ms (vector only)
  • P95 hybrid latency: < 200ms
  • P99 with re-rank: < 500ms
  • Throughput: 1000 QPS per search unit
  • Index size: up to 6B vectors per service
  • Replicas: 3 for HA + load balancing
  • Partitions: Scale horizontally as needed
Evaluation

Retrieval Quality Metrics

  • NDCG@10: target ≥ 0.80
  • MRR@10: target ≥ 0.75
  • Recall@10: target ≥ 0.85
  • Precision@5: target ≥ 0.70
  • Faithfulness score: ≥ 0.85 (RAGAS)
  • Answer relevance: ≥ 0.80 (RAGAS)
  • Continuous eval: RAGAS + custom evals

Security Architecture

Zero-trust, defense-in-depth across identity, network, data, and AI layers.

Identity (Zero Trust)

Authentication & Authorization

  • Entra ID (AAD) as identity provider
  • MFA enforced for all human identities
  • Managed Identities for all service auth
  • No service principals with passwords
  • Conditional Access: device compliance + location
  • PIM: JIT elevation for privileged roles
  • 6 custom RBAC roles (Reader to Admin)
  • Token lifetime: 1h access, 8h refresh
Network

Network Security

  • All services on private endpoints (no public IP)
  • Azure Private DNS zones for resolution
  • NSGs on all subnets (allow-list only)
  • Azure Firewall Premium (IDPS enabled)
  • DDoS Protection Standard on front-end
  • WAF on APIM (OWASP ruleset)
  • VNet peering to on-prem (ExpressRoute)
  • No internet egress except approved FQDNs
Data Protection

Encryption & Data Security

  • Encryption at rest: AES-256 (CMK)
  • Encryption in transit: TLS 1.3 minimum
  • CMK per tenant in Azure Key Vault (HSM)
  • Key rotation: automatic 90-day cycle
  • Double encryption: infra + service layer
  • Purge protection on all Key Vaults
  • Soft delete: 90-day retention minimum
  • DLP policies: Purview auto-classify + block
AI-Specific Security

AI / LLM Security Controls

  • Prompt injection detection (custom classifier)
  • Jailbreak attempt logging + blocking
  • Response content filtering (Azure Content Safety)
  • Input/output token auditing
  • Model access via private endpoint only
  • No training data leakage (PTU isolation)
  • PII scrubbing before LLM context assembly
  • Grounding-only policy: no unsupported claims
Compliance

Regulatory Compliance

  • ISO 27001 / 27017 / 27018 aligned
  • SOC 2 Type II (via Azure attestation)
  • GDPR: Data residency, right-to-erasure
  • HIPAA: Optional BAA, PHI handling
  • FedRAMP: Azure Gov region option
  • NIST CSF: Full framework alignment
  • Microsoft SSPA (supply chain)
Incident Response

Threat Detection & Response

  • Defender for Cloud (CSPM + CWP)
  • Microsoft Sentinel (SIEM/SOAR)
  • Diagnostic logs: all services → Sentinel
  • Anomaly detection: UEBA via Sentinel
  • IR playbooks: automated remediation
  • PEN test: annual + after major changes
  • MTTR target: < 4 hours (critical)

AI Guardrails Framework

Multi-layer controls ensuring safe, accurate, and policy-compliant AI outputs.

Input Guardrails (Pre-LLM)

Prompt Safety

Input Validation

  • Max input length: 8192 tokens
  • Prompt injection classifier (BERT-based)
  • Jailbreak pattern detection (rule + ML)
  • Profanity / hate speech filter
  • PII detection in user query (Presidio)
  • Disallowed topic classifier (configurable)
Context Assembly

RAG Context Controls

  • Max context window: 16K tokens
  • Source access check (RBAC) per chunk
  • PII scrub from retrieved chunks
  • Classification-level enforcement
  • Confidence threshold: discard < 0.5 score
  • Diversity: max 3 chunks per source doc

Output Guardrails (Post-LLM)

Hallucination Detection

Response Grounding

  • Faithfulness score via RAGAS ≥ 0.85
  • NLI entailment check vs. retrieved sources
  • Citation verification (every claim linked)
  • Out-of-context claim detection
  • Low-confidence flagging to end-user
Content Safety

Output Filtering

  • Azure Content Safety API (all responses)
  • PII detection in output (Presidio)
  • Harmful content categories: 4 severity levels
  • Legal/compliance disclaimer injection
  • Confidential info watermark stripping

    Implementation Roadmap

    Phased delivery over 6 months, from foundation to enterprise-grade production.

    Weeks 1–3 — Foundation

    Phase 0: Infrastructure & Identity

    Azure landing zone, VNet, private endpoints, Key Vault, Entra ID RBAC, Azure Policy baseline, tagging strategy, Terraform/Bicep IaC. All services deployed in private networking.

    Weeks 4–8 — Core Pipeline

    Phase 1: Ingestion & Vectorization

    Event Hubs + ADF connectors for Blob and SharePoint. Azure AI Document Intelligence integration. Chunking service deployment. Embedding pipeline (Azure OpenAI). AI Search index creation. Basic hybrid search functional.

    Weeks 9–12 — RAG API

    Phase 2: Retrieval & Generation

    Semantic Kernel orchestrator. GPT-4o PTU deployment. Hybrid retrieval with re-ranking. Prompt management service. Response grounding + citations. APIM gateway. Initial guardrails (content safety + PII).

    Weeks 13–16 — Security & Governance

    Phase 3: Enterprise Hardening

    Full guardrails framework. Microsoft Purview onboarding + data classification. Sentinel SIEM integration. Compliance Manager configuration. Audit logging. PIM + conditional access. Penetration testing.

    Weeks 17–20 — Applications & Multi-tenancy

    Phase 4: Application Layer

    Teams Copilot app. Admin portal (React + Static Web Apps). Multi-tenant isolation (index-per-tenant for sensitive). Developer API with OpenAPI spec. SDK publishing (Python + TypeScript). User feedback loop.

    Weeks 21–24 — Optimise & Scale

    Phase 5: Production Excellence

    RAGAS continuous eval pipeline. Performance tuning (caching, PTU sizing). Multi-region DR setup. FinOps dashboards. Load testing (10K concurrent). SLA sign-off. Runbook and ops playbooks. Go-live.
